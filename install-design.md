# Install Script Design

## What the install script does

1. Copy `src/` to `/opt/foldmount/src/`
2. Generate `/usr/local/bin/foldmount` with a hardcoded absolute path
3. Create `/var/lib/foldmount/` (data directory)
4. Check dependencies (`python3`, `e2fsprogs >= 1.45`)
5. Support `--uninstall` to reverse cleanly

## File layout after install

```
/
├── usr/local/bin/
│   └── foldmount              ← generated by install.sh
│
├── opt/foldmount/src/
│   ├── main.py
│   ├── cli.py
│   ├── commands.py
│   ├── startup.py
│   ├── paths.py
│   ├── image_cache.py
│   ├── volume_cache.py
│   ├── info_states.py
│   └── bash/
│       └── version_check.bash
│
└── var/lib/foldmount/
    ├── session                  ← written at runtime
    └── *.img                    ← created at runtime
```

## The two foldmount files

Repo (`foldmount`) — relative path, for local dev:
```bash
sudo python3 "$(dirname "$0")/src/main.py" "$@"
```

Installed (`/usr/local/bin/foldmount`) — absolute path, generated by install.sh:
```bash
sudo python3 /opt/foldmount/src/main.py "$@"
```

The installer writes the second file fresh. It never modifies the repo file.

## Dev workflow

- **Local dev**: run `./foldmount` from the repo as usual
- **Deploy to system**: run `sudo ./install.sh`
- **After changes**: re-run `sudo ./install.sh` to sync `/opt/foldmount/src/`

The source files are path-agnostic. Only the generated entry point knows the install location.
